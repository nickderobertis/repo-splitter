repo_splitter.main.remove_from_repo_history(repo_source, drop_files, github_token, keep_backup=True, auto_push_remove=False, backup_dir=None, follow_renames=True)[source]¶

Remove the passed files from the repo history entirely

Parameters

  • repo_source¶ (str) – clone url (remote) or file path (local) of repo that should be split

  • drop_files¶ (Sequence[str]) – files to be dropped in the repo history

  • github_token¶ (str) – personal access token for Github

  • keep_backup¶ (bool) – whether to retain a backup of the original repo in case something went wrong in removing history

  • auto_push_remove¶ (bool) – pass True to avoid prompt for whether to push the original repo with history removed

  • backup_dir¶ (Optional[str]) – pass file path to put backup of old repo there, otherwise uses repo_dest

  • follow_renames¶ (bool) – Whether to track previous names of files from the history and also include those

Returns

repo_splitter.main.split_repo(repo_source, repo_dest, new_repo_name, keep_files, github_token, all_branches=False, include_tags=False, remove_files_from_old_repo=True, keep_backup=True, auto_push_remove=False, backup_dir=None, follow_renames=True)[source]¶

Splits an existing Git repository into two repositories by selecting which files should be split into a new one.

Parameters

  • repo_source¶ (str) – clone url (remote) or file path (local) of repo that should be split

  • repo_dest¶ (str) – folder in which the new repo should be placed

  • new_repo_name¶ (str) – name for the new repo

  • keep_files¶ (Sequence[str]) – files to be kept in the new repo

  • github_token¶ (str) – personal access token for Github

  • all_branches¶ (bool) – whether to include all branches in the new repo or only the primary (remote)/active (local) one

  • include_tags¶ (bool) – whether to keep tags from the old repo in the new one

  • remove_files_from_old_repo¶ (bool) – whether to remove the split files and history from the original repo

  • keep_backup¶ (bool) – whether to retain a backup of the original repo in case something went wrong in removing history

  • auto_push_remove¶ (bool) – pass True to avoid prompt for whether to push the original repo with history removed

  • backup_dir¶ (Optional[str]) – pass file path to put backup of old repo there, otherwise uses repo_dest

  • follow_renames¶ (bool) – Whether to track previous names of files from the history and also include those
