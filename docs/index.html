Welcome to Repo Splitter documentation!¶

This is a Python and CLI tool for splitting repos. If you have a part of a repo which you would now like to be a separate repo, and you would like to separate the history of the two repos such that the new repo contains commits referencing its files and any references to those files in the old repo have been removed, then this is the tool for you.

An overview¶

These are the Python commands. For examples and CLI usage, see Getting started with repo_splitter.

repo_splitter.main.split_repo(repo_source, …)

Splits an existing Git repository into two repositories by selecting which files should be split into a new one.

repo_splitter.main.remove_from_repo_history(…)

Remove the passed files from the repo history entirely

repo_splitter.main.restore_from_backup(…)

Restores a repo to original after running split_repo or remove_from_repo_history
